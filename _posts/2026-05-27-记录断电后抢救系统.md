* 尝试对分区进行修复，考虑到我分区时主要使用btrfs和fat32这两个分区，因此检查和修复命令有些不同。

​	1.针对fat32（也就是我用来挂载/boot的分区）

```zsh
fsck.vfat -n /dev/nvme0n1p1 # 检查，模拟修复
fsck.vfat -a /dev/nvme0n1p1 # 自动修复
# 如果不行的话可以用更激进方式或者格了
```

​	2.针对btrfs（我的数据盘和根目录）

```zsh
btrfs check /dev/sda1 # 数据盘，所以基本上不会怎么写入什么数据的
btrfs check /dev/nvme0n1p2 # 出现错误：unknown key found in leaf

# 尝试修复
mount -t btrfs -o ro,recovery,usebackuproot /dev/nvme0n1p2 /mnt/fix # 可以先备份数据，当然我并没有这么做。备份完记得umount再往下面走。

mount -t btrfs -o usebackuproot /dev/nvme0n1p2 /mnt/fix # 如果这一步顺利和我一样的话，你可以顺着这个命令往下走。
btrfs scrub start /mnt/fix # 后台命令，进度用watch看btrfs scrub status /mnt/fix

# 如果极为不幸挂载不上来，你可以试试这个。这些是AI的，没有做测试。
sudo btrfs rescue zero-log /dev/sdXY # 删日志树
sudo btrfs check --repair /dev/sdXY # 深度修复，激进修复方式
```

* 重新grub-install安装grub

```zsh
# 前面记得/umount
mount -t btrfs -o subvol=@ /dev/nvme0n1p2 /mnt
mount --mkdir /dev/nvme0n1p1 /mnt/boot

# 进入容器
arch-chroot /mnt

# 安装grub
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

# 退出和重启
exit 
reboot
```
